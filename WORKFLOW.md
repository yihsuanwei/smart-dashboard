# 📋 開發工作流程

## 標準功能開發流程

每次完成功能開發後，請按照以下步驟進行：

### 1️⃣ 完成功能開發
- 實作新功能或修復 bug
- 測試功能是否正常運作

### 2️⃣ 更新 README.md
- 在 `## 🆕 版本更新歷史` 區塊最上方新增版本記錄
- 使用當天日期（格式：YYYY-MM-DD）
- **撰寫原則**：
  - 非常簡要，每個功能一句話
  - 不超過 50-100 字
  - 只記錄核心功能，不提及顏色、UI、位置等細節
  - 格式：`- 功能描述，說明重點`

**範例**：
```markdown
### 版本 3.1.5 (2025-11-27)

**📈 Performance Dashboard**
- 新增 TACOS Trend 折線圖，支援 2024 vs 2025 年度對比
```

### 3️⃣ 總結更新內容
完成 README 更新後，提供以下格式的總結給使用者：

**格式**：
```
YYYY-MM-DD: 更新主題
- 功能點 1
- 功能點 2
```

**排序**：從最新日期開始，倒序排列到最早

**字數限制**：每個日期的更新說明不超過 100 字

### 4️⃣ 使用者發布更新
使用者會將總結內容分享給團隊

---

## 快速檢查清單

每次完成功能時：
- [ ] 功能已開發並測試
- [ ] README.md 已更新（版本歷史區塊）
- [ ] 已提供簡潔的更新總結（倒序排列）
- [ ] 總結字數控制在 100 字內

---

## 注意事項

1. **簡潔原則**：只寫核心功能，避免技術細節
2. **用戶視角**：從使用者角度描述功能價值
3. **一致格式**：保持版本記錄格式統一
4. **日期正確**：使用當天實際日期
